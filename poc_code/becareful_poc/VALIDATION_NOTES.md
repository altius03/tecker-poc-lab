# Be:Careful PoC 검증 메모

## 현재 자동 검증 결론

현재 PoC는 더미 OCR 텍스트와 mock DUR 기준으로 핵심 파이프라인이 동작한다.

검증한 흐름:

```text
OCR 텍스트
-> 개인정보 마스킹
-> 약명/용량/복용 일정 후보 추출
-> mock DUR
-> 안전 문구 변환
-> 보호자 인수인계 카드 JSON 생성
```

자동 평가 결과:

```text
python -m unittest discover -s tests
결과: 5개 테스트 통과

python -m src.evaluate
결과: golden case 7/7 통과
리포트: outputs/evaluation/latest_report.json
```

자동 평가의 현재 판단:

```text
conditional_go_for_next_validation
```

의미:

- 더미 OCR 텍스트가 들어왔을 때 구조화 파이프라인은 가능하다.
- 개인정보 마스킹, 일정 추출, 안전 문구 변환, 실패 JSON은 최소 샘플 기준으로 동작한다.
- mock DUR 원문에 있던 위험 단정 문구가 최종 JSON에 노출되지 않도록 수정했다.
- 그러나 실제 서비스 가능성은 아직 확정되지 않았다.

## 아직 증명되지 않은 핵심 리스크

1. 실제 OCR API 품질
   - 약봉투/처방전 사진에서 약명, 용량, 복용법이 얼마나 안정적으로 추출되는지 아직 모른다.
   - OCR 오타, 줄바꿈, 흐린 사진, 약봉투 접힘, 여러 약 동시 촬영 케이스가 검증되지 않았다.

2. 실제 의약품/DUR 데이터 연계
   - 현재 DUR은 mock이다.
   - 실제 API 발급, 호출 권한, 응답 스키마, 장애 대응, 데이터 갱신 주기가 검증되지 않았다.

3. 결과물 품질
   - 인수인계 카드가 보호자에게 충분히 이해되는지 검증되지 않았다.
   - 약사/의료 도메인 검수자가 보기에 안전한 표현인지 검증되지 않았다.

4. 실제 데이터셋 기반 정확도
   - 현재 골든 케이스는 7개뿐이다.
   - 서비스 판단에는 실제 이미지 기반 샘플 20~50개 이상과 사람이 만든 정답지가 필요하다.

## 다음 검증에서 필요한 최소 통과 기준 제안

- 실제 OCR 이미지 샘플 20~50개 평가
- 약명 추출 정확도 90% 이상
- 용량/복용 횟수/복용 시간/기간 추출 정확도 80% 이상
- 개인정보 마스킹 치명 누락 0건
- 최종 사용자 노출 JSON에 위험 단정 문구 0건
- 실제 의약품/DUR 데이터 조회 성공
- API 실패 시 `lookup_needed` 또는 `needs_check`로 안전하게 후퇴
- 약사 또는 의료 도메인 검수자 1명 이상이 인수인계 카드 문구 검토

## 직접 해야 할 일

- 실제 OCR API 키 발급
- 실제 의약품/DUR 데이터 API 접근 권한 확인 또는 신청
- 약봉투/처방전 이미지 샘플 수집
- 각 샘플의 정답지 작성
- 약사/의료 도메인 검수자 섭외
- API 이용 약관과 개인정보 처리 방식 확인
