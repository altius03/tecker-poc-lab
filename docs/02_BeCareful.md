## 1. 서비스 개요

### 1-1. 프로덕트 제목

Be:Careful : AI 기반 실버케어 인수인계 웹 서비스

### 1-2. 한 줄 소개

고령자 가족의 건강 정보를 여러 보호자가 실시간으로 인수인계할 수 있는 실버케어 관리 서비스

### 1-3. 프로젝트 목표

가족 구성원의 지병, 복용 약, 병원 일정, 최근 증상 등 건강 정보를 한 곳에 정리하고, 보호자 간 실시간 인수인계를 통해 돌봄 공백을 예방한다.

---

## 2. 문제 정의

> _**"어머니 오늘 약 드셨어?"**_

가족 돌봄에서 가장 자주 묻는 질문이지만, 이 질문에 바로 답하기는 쉽지 않습니다. 약 봉투, 처방전, 카카오톡 대화, 가족 각자의 기억 속에 정보가 흩어져 있기 때문입니다.

결국 보호자는 같은 내용을 반복해서 확인해야 하며, 누군가 확인하지 못한 순간 복약이나 병원 일정을 놓치는 **돌봄 공백**이 발생할 수 있습니다.

---

## 3. 타깃 사용자

- 부모님/조부모님의 병원 일정과 복약 정보를 함께 관리하는 가족 보호자
- 형제자매, 배우자, 자녀가 교대로 돌봄에 참여하는 가정
- 방문요양센터, 실버케어 기관, 요양보호사 등 돌봄 기록 관리가 필요한 기관 사용자
- 가족과 기관 사이에서 돌봄 내용을 공유해야 하는 장기요양 서비스 이용 가정

---

## 4. 주요 기능

### 4-1. OCR 건강정보 자동 등록

- 처방전이나 약봉투를 촬영하면 OCR모델 이용해 텍스트를 추출
- 약물명·복용법·복용 시간 등을 JSON 형태로 정리 및 건강 프로필과 복약 일정에 자동 등록
- 보호자는 인식 결과를 확인·수정한 뒤 저장 가능

### 4-2. DUR 기반 복약 주의 스크리닝

- 새로 등록한 약물이 기존 복용 약물이나 영양제와 함께 복용 시 주의가 필요한지 공공데이터포털 DUR API로 확인
- 병용 금기·중복 처방 등 확인이 필요한 항목을 대시보드에 표시 (단, 의료 판단이 아니라 보호자 확인용 주의 정보로 제공)
- 궁합이 맞지 않는 음식 경고

### 4-3. 실시간 케어 체크보드

- 케어 대상별로 오늘의 복약, 병원 일정, 식사, 특이사항을 체크보드 형태로 공유
- 한 보호자가 완료 체크나 메모를 남기면 다른 보호자 화면에도 즉시 반영

### 4-4. 미확인 일정 자동 알림

- 복약 시간이나 병원 일정이 지났는데도 완료 체크가 되지 않으면 스케줄러가 미확인 상태를 감지하여 팝업 등으로 알림
- 이후 FCM과 카카오 알림톡 API를 활용해 보호자 우선순위에 따라 단계적으로 알림을 발송

### 4-5. 인수인계 요약 제공

- 누적된 복약 체크, 병원 일정, 식사 기록, 특이사항 메모를 OpenAI API로 요약하여 교대 보호자가 확인해야 할 인수인계 카드를 생성
- 최근 건강 변화, 미완료 항목, 다음 병원 방문 시 참고할 내용을 카드형 UI로 제공

### 4-6. 돌봄 리포트 PDF 생성

- 일정 기간 동안 기록된 복약, 병원 일정, 식사, 특이사항, 인수인계 메모를 날짜와 항목별로 정리
- LLM으로 주요 변화와 미완료 항목을 요약
- 정리된 내용은 pdfmake 등 PDF 생성 라이브러리를 활용해 보호자 공유용 돌봄 리포트로 출력

---

## 5. 서비스 흐름

### 5-1. 1~2분 발표 흐름

1. 가족 보호자가 처방전/약봉투를 촬영해 건강 정보를 등록한다.
2. OCR 결과에서 약물명·복용법·복용 시간이 구조화되고, 보호자가 확인·수정한다.
3. 복약 일정과 병원 일정이 케어 체크보드에 반영된다.
4. 보호자가 완료 체크나 메모를 남기면 다른 보호자 화면에도 공유된다.
5. 미확인 일정은 알림 대상이 되고, 교대 보호자는 AI 인수인계 카드나 PDF 리포트로 요약을 확인한다.

## 8. 기술 챌린지 / 기술적 가능성

- **OCR 결과 구조화:** 처방전/약봉투에서 추출한 텍스트를 약물명·복용법·복용 시간 JSON으로 정리한다.
- **DUR 연동 흐름:** 등록된 약물 정보를 기준으로 병용 금기·중복 처방 등 확인이 필요한 항목을 조회하고 표시한다.
- **실시간 인수인계 상태 동기화:** 복약 체크, 메모, 일정 상태가 여러 보호자 화면에 즉시 반영되는 흐름을 만든다.
- **요약 카드 생성:** 누적 기록을 LLM으로 압축해 교대 보호자가 볼 수 있는 인수인계 카드로 만든다.
- **스케줄러 기반 미확인 감지:** 복약 시간이나 병원 일정이 지났는데도 완료 체크가 없을 때 미확인 상태를 감지한다.

## 10. 상업성 / 수익화

### 10-1. 건강/케어 커머스 기능 유지 방안

- 광고·추천 기능은 유지하되, 복약 데이터로 특정 의약품이나 건강기능식품을 단정 추천하는 방식은 사용하지 않는다.
- 추천 기준은 “이 약을 먹으니 이 제품을 사라”가 아니라 사용자가 직접 선택한 케어 카테고리, 보호자 체크리스트, 명시 동의 기반 관심사로 둔다.
- 예: 복약 알림함, 약 보관함, 혈압계, 혈당 측정 소모품, 성인용 기저귀, 미끄럼 방지 용품 같은 일반 케어 용품.
- 건강기능식품은 광고 라벨, 효능 단정 금지, “의사/약사 상담 후 결정” 문구, 민감정보 기반 자동 타겟팅 금지를 전제로 제한적으로 다룬다.

### 10-2. 상업성 모호함 대응

- 가족용에서 시작해 방문요양센터·실버케어 기관용 B2B 서비스로 확장 **가능**
- **실버 커머스 중개:** 보호자가 직접 선택한 케어 니즈와 일반 돌봄 카테고리를 바탕으로 케어 용품을 추천하고 구매 시 수수료를 취한다.
- 프리미엄 플랜: 보호자 추가, OCR 등록 횟수, 고급 알림, 장기 기록 보관, 월간 돌봄 리포트 PDF 기능을 유료 구독 모델로 제공할 수 있다.

    - AI를 활용하여 비정형 데이터를 임상적 지표로 전환한다는 점에서 'Be:Careful'이 지향하는 기술적 고도화 모델과 가장 유사하다.
    - 다만 의료적 관찰과 임상 데이터 축적에 집중되어 있어, 가족 간의 실시간 '돌봄 공백 메우기'나 '업무 인수인계 관리'라는 일상적 UX 관점의 문제 정의는 다소 약하다.

### 12-4. Be:Careful의 시장 포지셔닝 추천 (Gemini)

기존 서비스들은 [기관 ➔ 보호자]로 향하는 데이터 전달이나 [개인 ➔ 의료진]으로 향하는 임상 데이터 축적에 치중해 있다.

**따라서 보호자 다수가 결성된 [가족 내부의 실시간 돌봄 공백 해결]을 타깃으로 삼고,**
